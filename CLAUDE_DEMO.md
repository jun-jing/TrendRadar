### 1.2. 未读消息优先处理规则
**核心理念**: 及时响应，维护良好沟通
```
1. **严格优先级**: 发现任何未读消息时，立即暂停其他操作
2. **全量处理**: 必须处理完所有未读消息，直到未读数量归零
3. **实时监控**: 每次操作前检查是否有新的未读消息
4. **循环检查**: 处理完一轮后重新检查，确保没有遗漏
5. **状态确认**: 只有在未读消息为0时，才能继续后续操作
```

### 1.3. 远程面试要求规则
**核心理念**: 远程面试优先，高效便捷沟通

**主要原因**:
- **时间成本考虑**: 避免长途通勤浪费时间，节省的时间可以用于参加更多面试机会
- **效率优化**: 远程面试安排更灵活，沟通效率更高

```
1. **强制要求**: 无论企业如何邀请，都要求远程面试
2. **话术策略**: 礼貌但坚持，主要强调时间成本和效率考虑
3. **灵活表达**: "希望能安排远程面试，主要是考虑时间成本，这样可以更高效地沟通"
4. **备选方案**: 如企业坚持现场，可协商视频初筛
5. **统一原则**: 所有沟通模板都要包含远程面试偏好
6. **距离判断**: 如果企业因距离原因且不提供远程面试，礼貌感谢并婉拒
7. **婉拒模板**: "感谢您的关注！由于地理位置限制且无法安排远程面试，暂时无法参与。祝您工作顺利，找到合适的候选人！"
```

## Claude 执行步骤

### 步骤1: 启动浏览器
```
使用 mcp__playwright__browser_navigate 导航到 Boss直聘聊天页面
```

### 步骤2: 扫描未读消息
```
使用 mcp__playwright__browser_snapshot 获取页面快照
识别带有未读标识的聊天窗口
按优先级排序未读消息
```

### 步骤3: 逐个处理消息
```
对每个未读消息:
1. 使用 mcp__playwright__browser_click 点击聊天窗口
2. 使用 mcp__playwright__browser_snapshot 查看对话内容
3. 分析企业地理位置和消息内容
4. 选择合适的回复模板
5. 使用 mcp__playwright__browser_type 输入回复
6. 使用 mcp__playwright__browser_click 发送消息
7. 等待 3-5 秒再处理下一个
```

### 步骤3.5: 主动搜索和沟通职位
```
完成未读消息后执行主动搜索:
1. 导航到职位搜索页面
2. 使用关键词组合搜索匹配职位
3. 分析职位JD，计算技能匹配度
4. 筛选急招和高匹配职位
5. 点击"立即沟通"按钮
6. 发送个性化主动沟通消息
7. 记录沟通职位，避免重复
```

### 步骤4: 智能回复逻辑
```
IF 企业不在武汉 THEN
    使用远程询问模板
ELSE
    根据消息内容选择本地沟通模板
END IF

消息内容关键词匹配:
- "面试" → 确认面试方式模板
- "技术"/"要求" → 询问技术栈模板
- "简历"/"了解" → 询问详情模板
- 其他 → 默认兴趣表达模板
```

## 安全机制

### 防重复发送
- 检查最后一条消息是否为自己发送
- 避免连续发送相同内容
- 确认发送成功后再继续

### 人工确认点
- 每处理 3 个消息后暂停，询问是否继续
- 遇到复杂对话时停止，等待人工介入
- 重要消息（如面试邀请）标记提醒

### 频率控制
- 每次回复间隔 3-5 秒
- 单次会话最多处理 10 个未读消息
- 检测到异常反馈时停止操作
