# 🔄 Git 分阶段提交工作流程规范

## 📋 核心原则
1. **原子化提交**: 每个提交只做一件事
2. **清晰记录**: 每次提交都有明确的目的说明  
3. **互动确认**: 重要修改需要人工确认

## 🎯 工作流程

### 1. 任务规划
- 使用 TodoWrite 创建任务清单
- 将大任务拆分为独立模块
- 每个模块对应一次提交

### 2. 提交审查
Claude 提供详细报告：
- 📝 修改摘要
- 📂 涉及文件列表  
- ✅ 主要改动点
- 💡 建议的提交信息

### 3. 用户确认后提交
```bash
git add [文件]
git commit -m "[类型]: 描述"
```

## 📝 提交类型规范

- `feat`: 新功能
- `fix`: 修复bug  
- `docs`: 文档更新
- `refactor`: 重构代码
- `chore`: 构建/工具链

## 🔔 提交模板

**修改摘要**: [一句话描述]  
**文件列表**: 
- file1.py (说明)
- file2.py (说明)

**主要改动**:
- 改动点1
- 改动点2

**建议提交信息**: `[类型]: 描述`

## ⚠️ 安全规则

- 永不提交敏感信息（API密钥、密码）
- 提交前检查：无调试语句、临时文件
- 每次提交保持原子化和清晰
